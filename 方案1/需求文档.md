## 1. 引言

### 1.1 目的

本文档旨在全面、准确地描述微波系统研发管理平台的功能需求、非功能性需求、接口需求和数据需求。本文档将作为系统开发、测试、验收的基准依据，确保所有利益相关方对系统需求有统一、清晰的理解。

### 1.2 范围

#### 1.2.1 系统边界

本系统是一个面向微波系统研发部门的综合管理平台，覆盖项目全生命周期管理、技术资产沉淀与复用、知识管理、团队协作等核心领域。

#### 1.2.2 包含内容

- 门户界面与工作台
- 用户管理与组织架构
- 项目管理与自动排程
- 项目开发与流程执行
- 产品货架与技术货架
- 知识库管理
- 技术论坛
- 即时通信

#### 1.2.3 不包含内容

- 财务管理系统（仅对接预算数据）
- 生产制造执行系统MES（仅触发流程对接）
- 企业资源计划ERP（仅触发流程对接）

### 1.3 定义与缩略语

| 术语/缩略语 | 定义 |
|------------|------|
| **IPD** | Integrated Product Development，集成产品开发 |
| **DCP** | Decision Check Point，决策评审点 |
| **CPM** | Critical Path Method，关键路径法 |
| **IPMT** | Integrated Portfolio Management Team，集成组合管理团队 |
| **BOM** | Bill of Materials，物料清单 |
| **MES** | Manufacturing Execution System，制造执行系统 |
| **ERP** | Enterprise Resource Planning，企业资源计划 |
| **L1/L2/L3/L4** | 流程层级，L1为最高层级，L4为具体活动层级 |
| **FS/SS/FF/SF** | 活动依赖关系类型：完成-开始、开始-开始、完成-完成、开始-完成 |

## 2. 总体描述

### 2.1 系统视角

#### 2.1.1 系统定位

本系统是微波系统研发部门的核心业务支撑平台，定位为：
- **项目管理中心**：统筹项目全生命周期
- **技术资产库**：沉淀和复用技术成果
- **知识管理平台**：系统化智力资产管理
- **协同工作空间**：支持团队协作与沟通

#### 2.1.2 系统架构

系统采用模块化架构设计，核心模块可独立开发、独立部署，支持：
- 模块间的松耦合集成
- 增量式功能扩展
- 多团队并行开发

```
┌─────────────────────────────────────────────────────────────┐
│                        门户界面层                              │
│              （部门首页、工作台、个人中心）                      │
├─────────────────────────────────────────────────────────────┤
│                        应用服务层                              │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ │
│  │用户管理 │ │项目管理 │ │项目开发 │ │产品货架 │ │技术货架 │ │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘ └─────────┘ │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐             │
│  │ 知识库  │ │技术论坛 │ │即时通信 │ │自动排程 │             │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘             │
├─────────────────────────────────────────────────────────────┤
│                        基础服务层                              │
│     （流程引擎、权限管理、文件管理、消息通知、Git集成）           │
├─────────────────────────────────────────────────────────────┤
│                        数据存储层                              │
│         （关系数据库、文档存储、文件存储、缓存）                 │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 用户特征

#### 2.2.1 用户分类

| 用户角色 | 描述 | 主要职责 | 预估数量 |
|---------|------|----------|---------|
| **系统管理员** | 平台运维人员 | 系统配置、用户管理、权限分配 | 2-3人 |
| **部门领导** | 部门负责人 | 项目审批、资源调配、决策评审 | 3-5人 |
| **团队组长** | 各团队负责人 | 团队管理、任务分配、进度把控 | 8-12人 |
| **设计师** | 核心研发人员 | 项目执行、设计开发、交付物产出 | 80-120人 |
| **其他人员** | 兼职/支持人员 | 综合管理支持、档案管理等 | 10-15人 |

#### 2.2.2 用户技能水平

- **技术能力**：具备微波系统、射频电路、数字电路等专业背景
- **软件能力**：熟练使用Altium Designer、ANSYS、CST、MATLAB等专业软件
- **IT能力**：具备基本的信息化系统使用能力

### 2.3 运行环境

#### 2.3.1 服务器环境

| 组件 | 要求 |
|------|------|
| **部署方式** | 局域网内部署，支持离线使用 |
| **操作系统** | Linux（推荐CentOS 7+ / Ubuntu 20.04+） |
| **Web服务器** | Nginx 1.20+ |
| **应用服务器** | 支持容器化部署（Docker/Kubernetes） |
| **数据库** | PostgreSQL 14+（主库）、Redis 6+（缓存） |
| **文件存储** | 本地存储/NAS（支持版本控制集成） |

#### 2.3.2 客户端环境

| 组件 | 要求 |
|------|------|
| **浏览器** | Chrome 90+、Edge 90+、Firefox 88+ |
| **分辨率** | 1920×1080及以上（推荐） |
| **本地软件** | Git客户端、Obsidian（可选）、各类设计软件 |

### 2.4 约束与假设

#### 2.4.1 技术约束

1. **离线运行**：系统必须支持局域网内部署，不依赖互联网
2. **Git集成**：文件版本管理必须集成Git
3. **MS Project兼容**：计划导入导出需支持MS Project格式
4. **专业软件调用**：支持调用本地专业设计软件

#### 2.4.2 业务约束

1. **IPD规范**：项目管理需符合IPD方法论要求
2. **DCP强制**：决策评审点不可跳过
3. **流程标准化**：L1-L4流程层级需严格执行

#### 2.4.3 假设条件

1. 用户具备基本的计算机操作能力
2. 局域网环境稳定可靠
3. 专业设计软件已预装在用户工作站上
4. Git服务器已部署并可访问

### 3.3 项目管理模块

#### 3.3.1 功能描述

项目管理模块是系统的核心模块，负责项目的全生命周期管理，包括项目创建、计划管理、资源管理、进度跟踪等功能。

#### 3.3.2 项目类别

| 类别代码 | 类别名称 | 说明 |
|---------|---------|------|
| PROD-SINGLE | 单机开发 | 完整单机产品开发 |
| PROD-MODULE | 模块开发 | 通用模块开发 |
| PROD-SOFT | 软件开发 | 嵌入式软件、应用软件开发 |
| TECH-DEV | 技术开发 | 通用技术开发 |
| PROC-DEV | 流程开发 | 业务流程定义与优化 |
| KNOW-DEV | 知识库开发 | 知识内容整理与沉淀 |
| SHELF-ADD | 产品上架 | 产品货架入库流程 |

#### 3.3.3 五步项目创建向导

| 步骤 | 步骤名称 | 核心内容 | 系统动作 |
|------|---------|---------|---------|
| 步骤1 | 基础信息录入 | 项目名称、代号、描述、起止时间、预算范围 | 名称查重、代号自动生成 |
| 步骤2 | 项目类别选择 | 选择项目类别 | 展示类别特征说明，推荐默认流程模板 |
| 步骤3 | 流程模板绑定 | 确认/选择/自定义流程模板 | 可视化预览流程图 |
| 步骤4 | 团队成员分配 | 分配产品负责人、项目负责人、技术负责人等 | 从组织架构选择人员，校验关键角色完整性 |
| 步骤5 | 创建确认执行 | 汇总展示配置信息，用户确认 | 创建项目记录、初始化流程实例、自动排程、创建本地文件夹、发送启动通知 |

#### 3.3.4 项目角色定义

### 3.4 项目开发模块

#### 3.4.1 功能描述

项目开发模块是团队成员日常工作的主战场，提供流程可视化执行、任务管理、交付物管理、协同工作等功能。

#### 3.4.2 流程活动执行

| 需求编号 | 需求名称 | 需求描述 | 优先级 |
|---------|---------|---------|--------|
| PD-001 | 流程全景视图 | 图形化展示完整开发流程，当前活动高亮标识 | 高 |
| PD-002 | 活动节点交互 | 点击查看任意活动的定义、输入输出交付物、执行指南 | 高 |
| PD-003 | 任务执行面板 | 任务认领、进度填报、交付物上传、完成确认 | 高 |
| PD-004 | 进度自动更新 | 综合交付物状态、工时消耗、检查项完成度计算进度 | 高 |
| PD-005 | 活动输入反馈 | 对活动输入有疑问可直接反馈给任务派发人员和领导 | 中 |
| PD-006 | 协同编辑 | 其他成员可查看并直接回复或修改活动输入 | 中 |
| PD-007 | 操作记录 | 记录所有操作过程以便后期查看 | 中 |

#### 3.4.3 交付物管理

| 需求编号 | 需求名称 | 需求描述 | 优先级 |
