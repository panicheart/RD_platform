1. 文档概述

文档目的

本文档详细定义了研发管理平台（RDP）的技术规格，包括系统架构、数据模型、API接口、功能规格、开发规范和验收标准。 本文档面向开发团队、测试团队和运维团队，作为系统开发和验收的技术依据。

技术栈规格

前端
  • React 18.x
  • TypeScript 5.x
  • Ant Design 5.x
  • Vite 5.x
后端
  • Go 1.22+
  • Gin 1.9+
  • GORM 2.x
  • Casbin 2.x
数据
  • PostgreSQL 16
  • Redis 7.x
  • MeiliSearch 1.x
  • MinIO

4. REST API接口规格

接口规范

GET /api/v1/users
获取用户列表
POST /api/v1/users
创建用户
PUT /api/v1/users/{id}
更新用户
DELETE /api/v1/users/{id}
删除用户

错误码定义

错误码 错误类型 说明
200SUCCESS成功
400BAD_REQUEST请求参数错误
401UNAUTHORIZED未认证
403FORBIDDEN无权限
404NOT_FOUND资源不存在
500INTERNAL_ERROR服务器内部错误

5. 模块功能规格

用户管理 (USER)

  • USER-001: 用户创建、编辑、删除
  • USER-002: 角色分配与管理
  • USER-003: 组织架构管理
  • USER-004: 个人主页与能力图谱

项目管理 (PROJECT)

  • PM-001: 项目创建向导
  • PM-002: 任务与里程碑管理
  • PM-003: 甘特图与进度跟踪
  • PM-004: MS Project导入导出

项目开发 (DEVELOPMENT)

  • PD-001: 流程全景视图
  • PD-002: 交付物管理
  • PD-003: 本地软件调用
  • PD-004: Git集成

产品货架 (PRODUCT)

  • PS-001: 产品展示与分类
  • PS-002: 成熟度评估(TRL)
  • PS-003: 相似创建
  • PS-004: 版本管理

7. 开发规范

项目结构规范

rdp/ ├── apps/ │ ├── web/ # 主前端应用(Shell) │ │ ├── src/ │ │ │ ├── api/ # API接口 │ │ │ ├── components/ # 公共组件 │ │ │ ├── views/ # 页面视图 │ │ │ ├── store/ # 状态管理 │ │ │ └── utils/ # 工具函数 │ │ └── package.json │ └── api/ # Go后端API │ ├── cmd/ # 入口文件 │ ├── internal/ │ │ ├── handler/ # 控制器 │ │ ├── service/ # 业务逻辑 │ │ ├── repository/ # 数据访问 │ │ └── model/ # 数据模型 │ └── go.mod ├── database/ # 数据库脚本 └── docs/ # 文档

编码规范

前端规范

  • • 使用TypeScript严格模式
  • • 组件使用PascalCase命名
  • • Hooks使用use前缀
  • • 使用Ant Design组件库
  • • Tailwind CSS用于自定义样式

后端规范

  • • 使用Go标准项目结构
  • • 接口使用小写+下划线
  • • 错误处理必须返回上下文
  • • 使用GORM进行数据库操作
  • • 日志使用Zap结构化输出
